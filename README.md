## 🗺 Дорожная карта проекта (Roadmap)

Проект разбит на 6 ключевых этапов: от проектирования до создания готового демонстрационного стенда.

### 🏁 Этап 1: Планирование проекта
- [x] Определение бизнес-целей (структурирование протоколов совещаний).
- [x] Формирование технических требований к качеству распознавания.
- [x] Составление архитектурной схемы системы (Audio -> STT -> LLM -> Protocol).

### 🔍 Этап 2: Исследование технологий (R&D)
- [ ] Сравнение библиотек для локального распознавания (STT):
    - `OpenAI Whisper` (разные размеры моделей).
    - `Vosk` (легковесность и скорость).
    - `Nvidia Riva` (высокая производительность).
- [ ] Исследование библиотек для **диаризации** (разделение голосов): `Pyannote.audio`.
- [ ] Выбор LLM для суммаризации текста (Llama 3, GigaChat API, YandexGPT или локальные модели).

### 🛠 Этап 3: Начало разработки (Core)
- [ ] Настройка среды окружения и Docker-контейнеров.
- [ ] Реализация модуля захвата и предварительной обработки аудио (очистка шума, нормализация).
- [ ] Интеграция выбранного STT-движка для преобразования речи в текст.

### 🚀 Этап 4: Рабочий прототип (MVP)
- [ ] Реализация базового разделения спикеров (Speaker 1, Speaker 2).
- [ ] Отработка пайплайна передачи "сырого" текста в LLM.
- [ ] Создание базового шаблона "Черновика протокола" (Дата, Участники, Темы, Решения).

### ✨ Этап 5: Готовый проект
- [ ] Оптимизация скорости обработки (инференса) для работы в реальном или около-реальном времени.
- [ ] Разработка интерфейса демонстрационного стенда.
- [ ] Финальная калибровка промптов для повышения точности протоколирования.

### 📊 Этап 6: Оформление результатов
- [ ] Тестирование на реальных записях внутренних совещаний.
- [ ] Сбор метрик точности (WER — Word Error Rate) и качества суммаризации.
- [ ] Подготовка финальной документации и видео-демонстрации работы стенда.
