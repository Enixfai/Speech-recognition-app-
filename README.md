# 🎙️ AI Meeting Assistant

![C++](https://img.shields.io/badge/C++-17-blue.svg)
![Qt](https://img.shields.io/badge/Qt-6.0+-41CD52.svg)
![Python](https://img.shields.io/badge/Python-3.10+-yellow.svg)
![AI](https://img.shields.io/badge/AI-Whisper%20%7C%20Llama3-orange.svg)

**AI Meeting Assistant** — это гибридное клиент-серверное приложение для транскрибации голосовых совещаний в реальном времени и автоматической генерации умных отчетов (протоколов).

Интерфейс написан на **C++ (Qt6)** для максимальной отзывчивости, а тяжелая логика искусственного интеллекта (распознавание речи и работа с LLM) вынесена в **Python-сервер**, общение с которым происходит через WebSockets.

---

## ✨ Ключевые возможности

* 🎤 **Распознавание в реальном времени (STT):** Эффект "живой печати" с использованием модели `faster-whisper`.
* 📝 **Генерация протоколов (LLM):** Автоматическое выделение тем, задач и решений с помощью Llama 3 (через Groq API).
* 🎵 **Поддержка файлов:** Загрузка и конвертация аудиофайлов (MP3, WAV, OGG) "на лету" с помощью `QAudioDecoder`.
* 🎨 **Современный UI:** Темная тема (Dark Mode) и поддержка рендеринга Markdown прямо в приложении.

---

## 🏗 Архитектура

* **Клиент (C++ / Qt6):** Захват аудио с микрофона (`QAudioSource`), управление интерфейсом (`QWidget`), стриминг байтов по сети (`QWebSocket`).
* **Сервер (Python):** Прием бинарных аудио-данных, VAD-фильтрация, распознавание текста и обращение к LLM-провайдеру.

## ⚙️ Установка и настройка (Ubuntu / Linux)

### 1. Системные зависимости
Убедитесь, что у вас установлены компилятор, CMake и библиотеки Qt6:
```bash
sudo apt update
sudo apt install build-essential cmake python3-venv python3-dev qt6-base-dev qt6-multimedia-dev qt6-websockets-dev libasound2-dev fonts-noto-color-emoji
```

### 2. Клонирование репозитория
```bash
git clone https://github.com/<Enixfai>/Speech-recognition-app-.git
cd Speech-recognition-app-
```

### 3. Настройка Python (Бэкенд)
Создайте виртуальное окружение и установите AI-библиотеки:
```bash
python3 -m venv venv
source venv/bin/activate
pip install faster-whisper websockets openai numpy
```

### 4. Сборка C++ интерфейса (Фронтенд)
```bash
mkdir build
cd build
cmake ..
cmake --build .
```

## 🚀 Запуск приложения

Для работы приложения необходимо запустить обе части: сервер и клиент. Вам понадобятся **два окна терминала**.

### Терминал 1: Запуск AI-сервера
Серверу требуется API ключ от [Groq](https://console.groq.com/) для генерации протоколов.
```bash
# Активируем окружение
source venv/bin/activate

# Задаем ключ (вставьте свой)
export LLM_API_KEY="gsk_ВАШ_КЛЮЧ..."

# Запускаем сервер
python3 scripts/scripts.py
```
*(При первом запуске сервер скачает веса модели Whisper — это может занять пару минут).*

### Терминал 2: Запуск интерфейса
```bash
cd build
./myapp
```

## 📖 Как пользоваться
1. В интерфейсе нажмите **«🔌 Подключиться»**.
2. Нажмите **«🎙 Старт микрофона»** и начните говорить. В окне в реальном времени будет появляться черновик текста.
3. По окончании мысли нажмите **«🛑 Стоп микрофона»**. Черновик зафиксируется в истории.
4. После остановки приложение автоматически запросит у нейросети создание **Протокола совещания**, который появится на экране в формате Markdown.
5. *Альтернативно:* Вы можете нажать **«📁 Загрузить MP3»**, чтобы отправить на сервер готовую запись.

## 🛠 Возможные проблемы
* **`ModuleNotFoundError: No module named 'websockets'`**: Вы забыли активировать виртуальное окружение (`source venv/bin/activate`) перед запуском сервера.
* **Connection refused**: Сервер на Питоне не запущен или еще не загрузил модель в память. Дождитесь надписи `Запуск WebSocket сервера`.
* **GLib-GObject-CRITICAL**: Стандартный ворнинг GStreamer в Ubuntu при работе с аудиофайлами. Не влияет на работу приложения.
